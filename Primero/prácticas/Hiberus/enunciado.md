# PRÁCTICA 1 — INTRODUCCIÓN A SCRUM Y TRABAJO ÁGIL

## Introducción

Actualmente muchas empresas tecnológicas utilizan metodologías ágiles para organizar el trabajo de sus equipos. Una de las metodologías más utilizadas es **Scrum**, que permite dividir tareas, organizar prioridades y coordinar el trabajo de varios desarrolladores.

En esta práctica el alumnado investigará los conceptos fundamentales de Scrum y realizará una pequeña simulación de trabajo en equipo similar a la utilizada en empresas tecnológicas reales.

---

# Objetivos

Con esta práctica se pretende que el alumnado:

- Comprenda qué es Scrum.
- Conozca los principales roles y eventos de Scrum.
- Aprenda cómo se organizan tareas en equipos tecnológicos.
- Utilice herramientas habituales del sector.
- Trabaje de forma colaborativa.

---

# PARTE 1 — DOCUMENTO INDIVIDUAL SOBRE SCRUM

Cada alumno deberá entregar individualmente un documento de texto mediante Google Classroom.

El documento deberá incluir los siguientes apartados.

---

## 1. ¿Qué es Scrum?

Explicar brevemente:

- Qué es Scrum.
- Para qué se utiliza.
- Qué ventajas tiene frente a otros métodos de trabajo.

---

## 2. Roles de Scrum

Explicar brevemente las funciones de:

- Product Owner
- Scrum Master
- Developers

---

## 3. Eventos de Scrum

Explicar brevemente:

- Sprint
- Sprint Planning
- Daily Scrum
- Sprint Review
- Retrospective

---

## 4. Herramientas utilizadas en empresas tecnológicas

Explicar para qué sirven herramientas como:

- Jira
- Trello
- GitHub
- Slack

---

# PARTE 2 — PROYECTO Y TABLERO KANBAN (GRUPAL)

Cada grupo deberá imaginar que forma parte de una empresa tecnológica y crear un pequeño proyecto ficticio.

---

## Posibles ejemplos

El grupo puede elegir cualquier idea sencilla, por ejemplo:

- Página web para una tienda.
- Aplicación de tareas.
- Sistema de incidencias.
- Web de reservas.
- Aplicación educativa.
- Biblioteca online.

---

## El documento grupal deberá incluir

### 1. Objetivo del proyecto

Explicar:

- Qué hace la aplicación.
- Qué problema intenta resolver.
- Qué tipo de usuarios la utilizarían.

---

### 2. Roles Scrum del grupo

El grupo deberá indicar:

- Quién será el Product Owner.
- Quién será el Scrum Master.
- Quiénes actuarán como Developers.

---

### 3. Lista de tareas

Crear varias tareas relacionadas con el proyecto.

Ejemplos:

- Diseñar pantalla principal.
- Crear login.
- Diseñar base de datos.
- Crear sistema de reservas.

---

### 4. Organización de tareas

Las tareas deberán organizarse según prioridad.

---

### 5. Tablero Kanban

El grupo deberá crear un pequeño tablero Kanban utilizando Trello, Jira o una herramienta similar.

El tablero deberá contener al menos:

- Pendiente
- En proceso
- Hecho

El documento deberá incluir una captura del tablero Kanban.

# PARTE 3 — SIMULACIÓN SCRUM

Cada grupo realizará una pequeña simulación en clase de aproximadamente 5 minutos.

La simulación deberá incluir:

## Daily Scrum

Cada integrante deberá explicar brevemente:

- Qué hizo.
- Qué está haciendo.
- Qué problemas tiene.

---

## Sprint Planning

El grupo deberá explicar:

- Qué tareas se han priorizado.
- Qué tareas se realizarán durante el Sprint.

---

# Preguntas posteriores

Tras la simulación se podrán realizar preguntas relacionadas con:

- Conceptos de Scrum.
- Roles.
- Eventos.
- Herramientas utilizadas.
- Organización del proyecto.
- Funcionamiento del tablero Kanban.
